**Beambox** brings Dockerfile semantics to [microsandbox](https://github.com/microsandbox/microsandbox). You describe a sandbox configuration once — either with a fluent builder or with a real `Dockerfile` — and beambox produces a reusable VM snapshot stored under `~/.microsandbox/images/`. Fresh microVMs spawn from that snapshot in milliseconds. No daemon, no registry.

## When to reach for it

- You want reproducible, prebuilt sandboxes for running untrusted code.
- You're already on microsandbox and you miss the ergonomics of a `Dockerfile`.
- You need millisecond cold-starts that container-based sandboxes can't hit.
